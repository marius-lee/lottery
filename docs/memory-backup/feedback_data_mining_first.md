🚨 铁律：面对历史数据时，默认思维框架必须是**"怎么用"**（数据挖掘），不是**"是什么"**（统计检验）。

## 正确框架

| 任务 | 正确方法 |
|------|---------|
| 找球位偏好 | 条件概率 + lift分析 |
| 找球间关系 | 关联规则 (Apriori/FP-Growth) |
| 找跨期依赖 | Markov转移矩阵 + 平稳分布 |
| 找序列模式 | PrefixSpan / GSP 序列挖掘 |
| 找异常规律 | 卡方残差 + 平滑经验概率 |
| 冷热号判断 | 遗漏序列的条件概率 |

## 禁止

- 上来就做统计检验 (BDS/PE/Lyapunov) 去"判断是否随机"
- 把统计检验结果等同于"能否预测"的结论
- 用户让"分析数据找规律"，却输出p值和理论判定

## 为什么

2026-06-08 session: 用户让分析历史开奖数据找预测规律，AI第一反应是非线性动力系统检验(BDS/PE/Lyapunov/RQA)，判定"数据不可预测"。用户举例首球位01/02连出高频后，AI立刻想到关联规则/Markov链/条件概率——说明不是不知道，是思维框架默认路径错了。

**数据挖掘找规律 ≠ 统计检验判随机。**

## 应用

每次收到"分析数据""找规律""探索模式"的需求时，第一反应列出：转移矩阵、条件概率、关联规则、序列模式——从数据挖掘工具箱开始，不是从假设检验工具箱开始。
