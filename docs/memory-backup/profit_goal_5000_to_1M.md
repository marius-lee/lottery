**北极星目标：初始投入 ¥5,000，6个月内实现盈利 ¥1,000,000（200倍回报）。**

这是项目存在的唯一目的。所有技术决策、功能优先级、架构选择、优化方向都必须围绕这个目标展开。

## 执行约束

每项修改、每个功能、每次优化前，必须回答以下问题：
1. 这项改动是否直接提升策略的赚钱能力？
2. 如果不做，会损失多少潜在收益？
3. 如果做，多久能体现在实盘结果上？
4. 是否在优化"看起来好"而非"能赚钱"的东西？

## 优先级法则

```
能直接提升收益 > 能降低风险/回撤 > 能加速策略迭代 > 能提升系统稳定性 > 代码美观
```

## 关键假设（需要持续验证）

- 5,000 元本金意味着只能做 A 股中小盘（高价股/大市值股买不起足够手数）
- 200 倍回报意味着必须捕捉高波动机会，夏普比率不是首要指标，绝对收益才是
- 6 个月时间窗口意味着低频策略（月度调仓）最多只有 6 次交易机会——这是不可能的，必须提高频率
- 路径依赖：5000→1万→2万→5万→20万→50万→100万，每阶段策略不同（本金小时必须激进，本金大后必须稳健）

## 禁止事项

- 禁止过度工程化（over-engineering）——功能够用即可
- 禁止为了"代码好看"花时间——除非影响实盘稳定性
- 禁止研究只有学术价值但无法实盘落地的因子/模型
- 禁止追求夏普比率而牺牲绝对收益

## 当前实盘障碍（优先解决）

1. 回测结果是否可信（交易成本、滑点、涨跌停、流动性）
2. 5,000 元能否买到策略选出的股票（单笔最低交易额 ~1000元，最多买 4-5 只）
3. 从信号到下单的全自动化链路
4. 实盘偏差监控

**How to apply:** 每次做决策时问自己："这个改动是让我离 100 万更近，还是更远？"
