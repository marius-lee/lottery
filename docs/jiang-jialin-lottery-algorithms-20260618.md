# 《抓住500万 彩票中奖绝招》 — 核心思想总结

**作者**: 蒋加林 (2001, 海天出版社)  
**提取日期**: 2026-06-18  
**来源**: OCR 扫描版 PDF, 约 200 页

---

## 核心理念

> "单个号码的出现与否是偶然的，毫无规律可言，但数个号码组成一组投注号码，是否合理，是否能中奖，则是有一定规律的。不同的号码组合，其中奖概率并不是相等的。"

— 不是预测号码，而是筛选"合理组合"。

---

## 八大绝招（乐透型彩票部分，与双色球直接相关）

### 第一绝招：小复式投注 (Smart Compound Betting)

**原理**: 选定 v 个号码后，不买全复式 C(v,6)，而是用预计算模板实现"二分之一复式"或"四分之一复式"覆盖。

- 16个号码全复式 = 11440注
- 四分之一复式模板 = 仅63注
- 保证：如果你的 v 个号选中6个红球，则至少中末等奖

**与现有系统关系**: `covering_design.py` 的 Steiner t-wise 覆盖就是此思路的现代版。已验证方向正确。

### 第二绝招：百万军中选大将 (Brute-Force Backtest) ⭐⭐⭐

**核心思路**:
1. 枚举所有 C(35,7) 组合（或过滤后的有效池）
2. 对过去 N 期逐一兑奖
3. 统计每注在 N 期内中奖次数
4. 选出"最少中K个号、最少中M次"的组合
5. 这些组合被称为"常胜将军"——不能保证中大奖，但中小奖频率最高

**书中的示例结果**:
- 11期窗口，19注投注
- 总投入 19×2×11 = 418元
- 总回报 3075元（四等奖1注+五等奖7注+六等奖19注+七等奖85注）
- 平均每注有5.89次中奖机会

**关键参数**:
- 参考期数窗口：选哪个时间段的数据做回测
- 最少中奖号码数 K：组合至少命中K个号才算"中奖"
- 最少中奖次数 M：在窗口中至少命中M期才算合格

**可集成性**: ⭐⭐⭐ 这是本书最独特的贡献。我们的系统生成号码后不做历史回测排名。可以将此作为一个新的选注模式：对有效池内所有组合做滑动窗口回测，选命中频率最高的N注。

**实现挑战**:
- 枚举全量 C(33,6)=1,107,568 组合 × N期兑奖 = 计算量较大
- 但可以限制到 valid pool 内（硬过滤后~85万组合）
- M1 8GB 内存可行：85万×4字节(bool) ≈ 3.4MB

### 第三绝招：相关性分析和捆绑式投注 (Correlation & Bundling) ⭐⭐

**原理**: 计算号码对的条件概率 P(B|A) = count(A∩B) / count(A)

```
例: A出现18次，A和C同时出现7次 → P(C|A) = 7/18 = 39%
    B出现17次，B和D同时出现8次 → P(D|B) = 8/17 = 47%
```

**捆绑投注**: 确定高频共现对(A,C)和(B,D)，选A时必须选C，选B时必须选D。其他号码自由组合。

**与现有系统关系**: 当前的 10 个分析模块（频率/遗漏/重号/邻号/012路/AC跨度/质数/龙风/同尾/相似期）都不含条件概率。这是真正的缺口。

**可集成性**: ⭐⭐ 可在 `static/js/analysis/` 中加一个"相关性分析"维度，用于用户手动选号参考。也可作为 `_build_pool()` 的一个软过滤因子。

### 第四绝招：相同码、斜角码、出现周期和分组号码投注

**相同码**: 与上期重合的号码（≈我们的"重号分析"）
**斜角码**: 走势图对角线位置的号码（≈我们的"邻号分析"）
**出现周期**: 号码遗漏期数（≈我们的"遗漏分析"）

→ 这三个我们已经覆盖。

### 第五绝招：分段号码统计和分段号码投注

将 1-33 分为若干段（如 01-10, 11-20, 21-33），统计每段出号数。将分段约束作为投注过滤条件。

→ 相当于我们的"三区分布"图表，但未用于过滤。可考虑作为硬/软过滤规则。

### 第六绝招：按位控制号码的出现范围

排序后第 1 位通常在 01-09，第 6 位通常在 28-33。限制每位号码范围来过滤无效组合。

→ 我们的 `_compute_luck_position_weights()` 已实现位置概率，但未做硬范围约束。**可加**。

### 第七绝招：基本参数控制号码的投注组合

三个基本参数：奇偶比、总码数（和值）、连号数。书中数据显示：
- 奇偶比：3:4/4:3 占 45%，2:5/5:2 占 45%，1:6 占 10%
- 总码数：集中在 100-142 范围
- 连号数：0-1 占 82%

→ 我们的软过滤只排除了 ≥5连号。奇偶比和和值范围可加为新的软过滤。

### 第八绝招：组合式投注和优化投注

用条件过滤减少投注组合数，与"旋转矩阵"思路类似。

→ 本质上就是我们的 hard/soft filtering + covering design。

## 与现有系统的对比

| 蒋加林绝招 | 现有系统覆盖 | 缺口 |
|-----------|-------------|------|
| 小复式投注 | covering_design.py ✓ | 无 |
| 百万军中选大将 | 无 | **历史回测排名算法** |
| 相关性分析/捆绑 | 无 | **条件概率 P(B\|A)** |
| 相同码/斜角码/周期 | 重号/邻号/遗漏 ✓ | 无 |
| 分段号码统计 | 三区分布图表 ✓ | 未做过滤 |
| 按位控制 | 位置权重(运气模式) ✓ | 未做硬范围 |
| 基本参数控制 | 软过滤(≥5连号) 部分 | **奇偶比/和值过滤** |
| 优化投注 | hard + soft filtering ✓ | 无 |

## 推荐集成优先级

### P0: 百万军中选大将（新选注模式）

最独特且现有系统未覆盖的思路。

实现方式：
- 新增 `selection_mode='backtest'` 参数
- 从 valid_pool 中对每个组合做滑动窗口兑奖
- 按历史命中频率排序，取 top-N
- 作为"生成号码"的一个可选项（类似贪心优化）

### P1: 相关性分析（新增分析维度 + 捆绑投注）

实现方式：
- 前端加一个"号码相关性"分析面板
- 计算所有 C(33,2)=528 对的 P(B|A)
- 显示 top 共现对
- 捆绑投注：用户选定锚定号码 → 系统推荐捆绑号码

### P2: 奇偶比 + 和值 + 按位范围（新增软过滤）

实现方式：
- `_check_soft()` 中加奇偶比检查和值范围检查
- `_check_hard()` 中加按位范围硬约束
