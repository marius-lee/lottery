## 方案 1: kyr0/lotto-ai — 一次性批量训练

- **来源**: https://github.com/kyr0/lotto-ai (JavaScript/TensorFlow.js)

### 训练方式

```
一次性批量训练。不是持续循环。

① 数据准备:
   lotto.csv → lotto.json (一次性)
   1000+ 期历史 → one-hot 编码
   
② 固定窗口切分:
   窗口=5期, 滑动生成训练样本
   train/val = 80%/20% (固定分割)

③ 训练:
   model.fit(X_train, Y_train, {
     epochs: 50,           ← 固定轮数
     batchSize: 32,        ← 固定批次
     validationSplit: 0.2, ← 固定验证集
     callbacks: [earlyStopping(patience: 5)]
   })
   
④ 预测:
   加载最佳模型 → 一次前向 → 输出概率 → 解码号码

⑤ 不循环。每次新数据来了, 手动重新 train。
```

### 算法
- Attention LSTM + Binary CrossEntropy
- Adam optimizer, Dropout + L2 reg
- one-hot 编码, 无 embedding 层

### 训练循环特点
| 维度 | 值 |
|------|:--:|
| 训练方式 | 一次性批量 |
| 是否持续 | 否 |
| 窗口大小 | 固定 5 期 |
| Epoch | 固定 50, early stop 容差 5 |
| 数据分割 | 80/20 固定 |
| 新数据处理 | 手动重新训练 |
| 验证集 | 固定分割, 不滑动 |

## 方案 2: KittenCN — 一次性批量训练 (双模型)

- **来源**: https://github.com/KittenCN/predict_Lottery_ticket (TensorFlow/PyTorch)

### 训练方式

```
一次性批量训练。红球蓝球分开。

① 数据准备:
   爬取 500.com → data.csv
   windows_size=3, 滑动窗口切分

② 数据集划分:
   训练 70% / 验证 15% / 测试 15%
   时间序列顺序划分 (不打乱)

③ 红球训练:
   red_epochs=60, batch_size=32
   LSTM+CRF, 负对数似然 loss
   每 epoch: 验证集评估 → 保存最佳模型
   早停: patience 逻辑

④ 蓝球训练:
   blue_epochs=20, batch_size=32
   LSTM+Softmax, CE loss
   同样的早停逻辑

⑤ 预测:
   加载最佳模型 → Viterbi解码 → 号码

⑥ 不循环。新数据来了手动重新训练。
   可同时训练多个 windows_size (3/5/7) 综合预测
```

### 算法
- Per-Ball LSTM + Global LSTM + CRF (红球)
- LSTM + Softmax (蓝球)
- Adam(lr=0.0005), SparseCategoricalCrossentropy

### 训练循环特点
| 维度 | 值 |
|------|:--:|
| 训练方式 | 一次性批量 |
| 是否持续 | 否 |
| 红蓝 | 分离, 各自独立训练 |
| 窗口大小 | 3 (可配, 多窗口综合) |
| Epoch | 红60 / 蓝20, 各有早停 |
| 数据分割 | 70/15/15 |
| 新数据处理 | 手动重新训练 |
| 验证集 | 固定分割, 时间序列顺序 |
| CRF约束 | Viterbi全局最优解码, 天然去重 |

## 方案 3: 微信综述 (三种范式) — 一次性批量训练

- **来源**: https://mp.weixin.qq.com/s?__biz=MzAwNDU1NzczMg==&mid=2247503317

### 训练方式

```
三种范式都是标准监督学习, 一次性批量:

范式 3a (位置独立分类):
  X=[N期历史 One-hot 拼接], Y=[6红位置各33类 + 1蓝16类]
  loss = Σ 各位置 CrossEntropy
  训练: fit() 一次性, epochs 可配

范式 3b (Seq2Seq):
  Encoder: 压缩 N 期历史 → 上下文向量
  Decoder: Teacher Forcing 逐位生成
  训练: fit() 一次性

范式 3c (49维伯努利):
  输入 N 期, 输出 49 维 0/1 标签
  训练: Binary CrossEntropy, 一次性批量
```

### 训练循环特点
| 维度 | 值 |
|------|:--:|
| 训练方式 | 一次性批量 |
| 是否持续 | 否 |
| 三种范式 | 各自独立, 不可切换 |
| 新数据处理 | 手动重新训练 |

## 关键差异分析

### 所有公开方案都是「一次性批量训练」

KittenCN、kyr0、微信综述全部是经典监督学习——训练完就完了。新数据来了手动重新 `train()`。**没有一个做持续循环自训练。**

### 我们独有的是「持续循环 + 自动更新」

这是从你的理念推导出来的创新——不是为了不同而不同，是为了:
1. 每轮不同超参数强迫模型从多角度学习
2. 新数据自动拉取 → 自动切目标 → 自动继续
3. 永远不停 → 永远在进化

### 可借鉴的点

| 来源 | 可借鉴 | 为什么 |
|------|--------|--------|
| KittenCN | **CRF 去重** | Viterbi 解码天然保证红球不重复, 不破坏依赖 |
| KittenCN | **红蓝分离训练** | 两种概率分布不应该混在一个模型里 |
| kyr0 | **固定窗口** | 可变窗口增加不稳定性, 可能固定反而更好 |
| 微信 3c | **49维伯努利** | 输出天然无重复, 训练简单 |
| EMNLP | **log尺度分桶** | 结构化中间表示 |
