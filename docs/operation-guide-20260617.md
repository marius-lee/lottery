## 控制区布局

```
┌─ 注数: [1|2|3] ───────────── 选 3 注即 ¥6/期
│
├─ ☐ 位置软过滤 ─────────────── 去 ≥5连号/间距≥24/新位置号
├─ ☐ 分散红球 ─────────────── Tier 1: 注间共享红球 ≤2，防重复浪费
├─ ☐ 贪心优化 ─────────────── Tier 2: 自动勾选分散红球，Jaccard 最大化注间距离
├─ ☐ 运气规则 ─────────────── 近 10 期位置热度偏置
│
├─ [生成号码] ──────────────── 主路径：池采样 + 选中选项
├─ [确定保存] ──────────────── 保存当前生成的号码
├─ [幸运开奖] ──────────────── 纯运气模式：位置加权独立抽号
└─ [覆盖设计] ──────────────── Tier 3: Steiner t-wise 覆盖 + 蓝球，返回覆盖率
```

## 三层策略说明

| 层 | 前端控件 | API 参数 | 原理 |
|----|---------|----------|------|
| Tier 1 | 分散红球 | `?max_overlap=2` | 采样循环内硬约束，候选注与已选注共享红球 >2 即拒绝 |
| Tier 2 | 贪心优化 | `?div=1&max_overlap=2` | 从 1000 候选池贪心选 N 注，每步最大化 min Jaccard 距离 |
| Tier 3 | 覆盖设计 | `/api/covering-diverse` | SA 引擎优化 Steiner t-wise 覆盖 + Laplace 蓝球分配 |

## 推荐操作路径

### 日常基础（¥6/期）
- 注数=3
- 不勾选任何选项
- 点「生成号码」
- 效果：硬过滤随机采样

### 日常分散（¥6/期）
- 注数=3
- 勾选「分散红球」
- 点「生成号码」
- 效果：3 注覆盖不重叠区域，避免 ¥4 买 ¥2 的覆盖

### 最优分散（¥6/期）
- 注数=3
- 勾选「贪心优化」（自动启用分散红球）
- 点「生成号码」
- 效果：max-min Jaccard 全局优化，比纯分散约束更优

### 冷号反弹（¥6/期）
- 注数=3
- 勾选「位置软过滤」
- 点「生成号码」
- 效果：排除历史上罕见出现的模式（≥5连号、间距≥24、从未出现过的位置号）

### 热度追踪（¥6/期）
- 注数=3
- 勾选「运气规则」
- 点「生成号码」
- 效果：近 10 期各位置热度加权，热号优先

### 覆盖保底（¥12/期）
- 注数=6
- 点「覆盖设计」
- 效果：Steiner t-wise 覆盖，若 6 红全在 15 热号内，≥覆盖率 概率命中 ≥t 红球
- 注意：生成较慢（~30 秒），SA 迭代 10000 次

## 开奖后操作

1. 点「开奖对比」面板
2. 点「对比最新开奖」按钮
3. 系统自动回测命中情况（红球命中数/蓝球是否命中）
4. 策略权重自动更新（James-Stein 收缩 + 滑动窗口）
5. 「复盘跟踪」面板可查看历史预测 vs 实际对比
