## 核心新算法：断区转换法 (Chapter 2) ⭐

### 算法原理

1. **行列分布表**: 将33个红球按6行×6列排列成矩阵
   - 第1列: 01,07,13,19,25,31
   - 第2列: 02,08,14,20,26,32
   - ...
   - 第6列: 06,12,18,24,30

2. **断区**: 每期有1-2行/列不出现中奖号码（称为断行/断列）
   - 断1列概率最高 → 排除该列5-6个号码 → 33选6变27选6
   - 断2列 → 33选6变21选6
   - 断3列 → 33选6变15选6
   - 最极端：断3行3列 → 只剩9个号码

3. **3D转换**: 将断列位置编码为3位数
   - 断0列 → 000
   - 断1列(第2列) → 002
   - 断2列(第3,6列) → 036
   - 断3列(第1,3,5列) → 135
   - 断4列以上只取前3位

4. **42注3D号码**: 所有断区3D号码共42种，分8个两码类型(00/03/06/13/14/25/36/A型)

5. **断区两码技术**(第二版新增): 将42注3D号按两码分组，通过排除/选择两码来进一步缩小范围

### 实战流程

1. 研判断列3D号码（百位/十位/个位分别选择）
2. 研判断行3D号码（同上）
3. 研判断区两码（缩小3D号码组合）
4. 确定的断行+断列区域 → 排除对应号码 → 剩余号码即为候选红球

### 压缩效率

| 断区情况 | 剩余号码 | 效果 |
|----------|----------|------|
| 断1行1列 | 22个 | 33→22 |
| 断2行1列 | 17个 | 33→17 |
| 断2行2列 | 15个 | 33→15 |
| 断3行3列 | 9个 | 33→9 |

## 可集成性评估

**不适合直接代码化**。断区转换法本质是人工看图研判流程——彩民观察走势图，判断下期断哪行哪列，需要图形化辅助。但**可以作为新功能面板**：在UI中展示行列分布表+断区3D号码走势，让用户手动研判后输入断区代码，系统据此过滤红球。

**复杂度级别**: 相当于微尔算法的量级，需要独立面板。
